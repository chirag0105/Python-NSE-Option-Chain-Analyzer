## Lessons Learned
- Translating the legacy algorithm directly into the backend `data_processor.py` was highly effective since we parsed changes linearly around the ATM slice offset, allowing us to port over the exact mapping behavior without recalculating loops inside the GUI.
- Utilizing WebSocket payloads incrementally for passing state snapshots limits DOM-repaint bottlenecks relative to having the browser infer state history internally. By having the backend store an active timeline array caching via `DataManager`, connecting clients immediately retrieve historical logs and are inherently resilient to single-frame desyncs.
